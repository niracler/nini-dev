# Design: Codex vs Claude 对比页面设计

## Context

将「AI 编程工具：三代范式」页面替换为更有实操价值的 Codex CLI vs Claude Code 对比页面。

**目标听众**：对 AI 了解甚少的物联网工程师
**信息密度**：极限单页，包含 5 个内容模块 + 6+ 脚注

---

## Goals / Non-Goals

**Goals:**

- 单页展示完整对比信息（性能 + 社区 + 行为 + 场景 + 结论）
- 遵循 design 规范：有信息量的视觉锚点，非纯文字装饰
- 酷炫但不失信息密度
- 脚注详细到位

**Non-Goals:**

- 不拆成多页
- 不做三代范式的铺垫

---

## 视觉锚点策略

本页的视觉锚点（全部是有信息量的元素）：

| 元素 | 类型 | 信息量来源 |
|------|------|-----------|
| 性能对比表 | 数据表格 | SWE-bench 真实数据 |
| 社区偏好数据 | 数据可视化 | Reddit 500+ 评论分析 |
| 行为特征对比 | 对比表格 | 社区总结的行为差异 |
| HN 精灵引用 | 引用块 | 记忆点，具体形象 |
| 场景适配表 | 对比表格 | 实操建议 |

**❌ 避免的伪视觉**：彩色卡片纯装饰、emoji 列表、空边框

---

## 布局方案

采用**极限密度单页**布局，使用 grid 系统：

```
┌─────────────────────────────────────────────────────────┐
│  标题 + 页面类型标注（右上角）                            │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────┐  ┌─────────────────────────┐  │
│  │ 性能对比表          │  │ 社区偏好数据            │  │
│  │ (代码质量/上下文/   │  │ 65.3% vs 34.7%         │  │
│  │  速度/交互风格)     │  │ + 胜出领域              │  │
│  └─────────────────────┘  └─────────────────────────┘  │
├─────────────────────────────────────────────────────────┤
│  行为特征对比表（指令遵循 / 执行反馈 / 典型问题）          │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────┐    │
│  │ 💬 HN 精灵引用（记忆点）                         │    │
│  └─────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────┤
│  场景适配 + 结论（横向紧凑排列）                          │
├─────────────────────────────────────────────────────────┤
│  脚注区域（6+ 来源）                                     │
└─────────────────────────────────────────────────────────┘
```

## 脚注设计

遵循详细脚注原则，6+ 来源：

| 序号 | 来源类型 | 内容 |
|------|---------|------|
| 1 | 🔬 L1 | SWE-bench Verified — 基准测试方法论 |
| 2 | 📊 L2 | Reddit 情绪分析 — 500+ 评论、偏好率计算方法 |
| 3 | 💬 L2 | HN Codex vs Claude 讨论 — 精灵引用来源 |
| 4 | 💬 L2 | HN GPT-5.1-Codex-Max 讨论 — 补充社区观点 |
| 5 | 📖 L2 | Builder.io 对比评测 — 实测视角 |
| 6 | 📖 L3 | 官方定价页面 — 成本参考 |

---

## 记忆点检查

- ✅ **核心记忆点**：「没有银弹：Codex 准确但慢，Claude 快但可能需返工」
- ✅ **形象类比**：「Codex 像字面意义的精灵」
- ✅ **视觉锚点**：多个数据表格，非纯文字
- ✅ **实操价值**：场景适配建议

---

## v-click 动画策略

考虑到信息密度极高，**节制使用 v-click**：

1. 性能表 + 社区数据：默认显示
2. 行为特征表：`v-click` 展开
3. 精灵引用：`v-click` 出现（制造记忆点）
4. 场景适配 + 结论：`v-click` 收尾

---

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| 单页信息过载 | 使用小字体 + 紧凑间距 + 合理 v-click |
| 表格太多视觉疲劳 | 配色区分 + 引用块打破节奏 |
| 脚注区域拥挤 | 精简描述，保留核心信息 |
