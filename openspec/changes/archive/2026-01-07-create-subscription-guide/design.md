# Design: AI 订阅指南

## Context

公司引入 AI 编程工具，每位开发同事有以下订阅额度选择：

| 套餐 | 月费 | 说明 |
|------|------|------|
| **$20 基础套餐** | 每个 ~$20 | codex/claude/gemini/cursor **可多选** + 国内模型 0-2 个 |
| **$100 进阶套餐** | ~$100 | codex/claude Max（不推荐新手直接选择） |

**$20 套餐示例**：

- Claude $20 + Cursor $20 + Kimi = 3 个服务
- Codex $20 + Gemini $20 = 2 个服务
- Claude $20 + Cursor $20 + Kimi + GLM = 4 个服务

**目标读者**：混合受众 — 从零基础新手到有一定经验的开发者

**前置条件**：

- 公司有美国代理节点
- 需要 Google 账号注册大部分服务

**配套资料**：作为 `ai-programming-share` PPT 的附录文档

## Goals / Non-Goals

**Goals:**

- 让任何技术水平的同事都能完成订阅
- 清晰标注各种订阅方式的风险等级
- 提供最新、可验证的订阅流程
- 解释公司订阅规则和分享机制

**Non-Goals:**

- 不推荐特定服务（仅提供信息，用户自行选择）
- 不提供代充/共享账号的具体联系方式
- 不承诺信息永久有效（需定期更新）

## 文档编写流程

> **核心约束：研究先行** — 每篇教程编写前必须先搜索验证最新信息，不能基于过时记忆编写。
>
> **硬约束：信息时效** — 所有价格、流程信息必须标注查证日期，超过 3 个月的信息需要重新验证。

**流程（每篇教程）：**

```text
┌─────────────────────────────────────────────────────────────┐
│  A-B 自动执行，C 处用户 Review                               │
│  信息过时或不满意 → 从 A 重来                                │
└─────────────────────────────────────────────────────────────┘

Step A: 研究准备（WebSearch + WebFetch）
        → 搜索最新订阅流程、支付方式
        → 验证当前价格和套餐
        → 收集常见问题和解决方案
        → 输出：研究摘要 + 关键信息来源
        ↓ (自动)
Step B: 编写初稿
        → 按文档规范编写
        → 分层结构：快速步骤 + 详细说明
        → 标注风险等级和信息来源
        → 添加常见问题 FAQ
        ↓ (自动)
Step C: 用户 Review ← 🔍 Review 点
        → 检查信息准确性
        → 确认风险标注合理
        → 决策：通过 / 补充信息 / 重写
```

**关键原则**：

- **信息溯源**：所有关键信息必须有来源链接
- **风险透明**：每种方式的风险必须明确说明
- **分层呈现**：快速上手 vs 详细教程并存
- **时效标注**：价格和流程信息标注查证日期

## Step A: 研究准备（WebSearch 必做）

每篇教程开始前，按内容类型进行针对性搜索：

| 内容类型 | 搜索示例 | 验证要点 |
|---------|---------|---------|
| 服务订阅 | `[服务名] subscription 2025 pricing` | 当前价格、支付方式 |
| 支付方式 | `[支付方式] 订阅 [服务名] 教程 2025` | 是否仍可用、具体步骤 |
| 常见问题 | `[服务名] subscription china issues` | 地区限制、常见报错 |

**搜索策略**：

1. **官方优先** — 先查官方定价页、帮助文档
2. **多源交叉** — 至少 2 个来源确认关键信息
3. **时效过滤** — 优先 2025 年的信息，谨慎使用 2024 年前的
4. **中文补充** — 搜索中文社区的实操经验（小红书、V2EX、知乎）

### 风险等级定义

| 等级 | 标记 | 定义 | 示例 |
|------|------|------|------|

## 各文档要点

### 00-快速指引.md

**目标**：5 分钟内帮助用户做出订阅决策

**内容**：

- 决策树：「你需要什么？」→ 推荐服务
- 套餐一览表（$20 可多选）
- 常见问题快速解答

### 01-前置准备.md

**目标**：零基础用户完成前置配置

**内容**：

- Google 账号注册（图文教程）
- 代理配置（公司节点使用说明）
- 常见问题：手机号验证、地区选择

**研究重点**：

- 当前 Google 账号注册是否需要手机验证
- 是否有地区限制

### 02-订阅方式.md

**目标**：全面介绍各种支付方式及风险

**内容**：

| 方式 | 风险 | 适用服务 | 说明 |
|------|------|----------|------|
| 海外银行卡 | 🟢 | 全部 | 最推荐，需要开通外币账户 |
| 国内 Visa/Mastercard | 🟢 | Cursor | 部分服务支持 |
| Apple 礼品卡 | 🟡 | iOS 应用内订阅 | 需要美区 Apple ID |
| 虚拟信用卡 | 🟡 | 全部 | Depay/Nobepay 等 |
| 淘宝代充 | 🔴 | 各类 | 可能被封号 |
| 共享账号 | 🔴 | 各类 | 不稳定，可能违规 |

**研究重点**：

- 虚拟信用卡服务的当前可用性
- 各服务对支付方式的限制

### 03-07 各服务订阅教程

**统一结构**：

- 当前价格和套餐
- 快速步骤（3 步）
- 详细教程（带截图占位）
- 支持的支付方式
- 常见问题 FAQ

**研究重点**（每个服务）：

- 当前定价
- 注册/订阅流程
- 地区限制
- 常见问题

### 08-套餐选择.md

**目标**：帮助用户选择合适的套餐组合

**内容**：

- $20 套餐推荐组合（可多选示例）
- $100 套餐适用场景
- 国内模型补充建议
- 「先尝试国外模型」原则说明

**套餐组合示例**：

| 组合 | 服务 | 月费合计 | 适用场景 |
|------|------|----------|----------|
| 入门型 | Claude $20 | $20 | 刚开始尝试 |
| 标准型 | Claude $20 + Cursor $20 | $40 | IDE 集成 + 独立对话 |
| 全面型 | Claude + Cursor + Gemini | $60 | 多模型对比使用 |
| 国内补充 | Claude + Cursor + Kimi | $40+ | 中文场景补充 |

### 09-订阅规则.md

**目标**：说明公司订阅规则

**内容**：

- 报销流程
- 每月分享要求
- 分享内容建议（可以用 LLM 辅助总结）

## 质量检查清单

每篇教程完成后检查：

- [ ] **信息时效**：价格和流程信息是否标注查证日期？
- [ ] **来源标注**：关键信息是否有来源链接？
- [ ] **风险标注**：支付方式是否标注风险等级？
- [ ] **分层结构**：是否有快速步骤 + 详细说明？
- [ ] **常见问题**：是否覆盖常见报错和解决方案？
- [ ] **截图占位**：需要截图的地方是否有占位标记？

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| 信息快速过时 | 标注查证日期，定期复查 |
| 高风险方式可能出问题 | 明确标注风险，用户自行决策 |
| 截图无法预先准备 | 使用占位符，后续补充 |
| 各服务流程可能变化 | 保持模块化，便于更新单篇 |

## Open Questions

- Apple 礼品卡支持 Claude 和 Codex 订阅
- 虚拟信用卡（Depay/Nobepay）对中国用户已不可用
- 需要准备视频教程

**待研究**：

- Apple 礼品卡具体充值流程（美区 Apple ID 创建 + 礼品卡兑换）
- 各服务最新定价和套餐变化
- 视频教程的制作工具和托管方式
