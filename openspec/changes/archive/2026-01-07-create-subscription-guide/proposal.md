# Change: 创建 AI 订阅指南

## Why

公司引入 AI 编程工具后，每位开发同事都有订阅额度，但存在以下问题：

1. **认知差距** — 很多同事对如何注册、订阅海外服务缺乏了解
2. **支付障碍** — 大部分海外服务不支持国内银行卡直接支付
3. **选择困难** — 面对 Claude、Codex、Gemini、Cursor 等多个服务，不知如何组合
4. **前置条件** — 部分同事可能连 Google 账号都没有，代理配置也不熟悉

需要一份系统性的订阅指南，作为 AI 编程分享 PPT 的配套资料。

## What Changes

### 新增能力

- **分层文档结构** — 快速指引（5 分钟上手）+ 详细教程（按需查阅）
- **风险等级标注** — 每种订阅方式标注 🟢低/🟡中/🔴高 风险
- **研究先行流程** — 每个教程编写前先搜索验证最新信息

### 新增文档

在 `ai-programming-share/docs/` 目录下新增订阅指南章节（10+ markdown 文件）：

| 文档 | 内容 |
|------|------|
| 入口页 | 快速概览 + 套餐推荐 |
| 前置准备 | Google 账号注册 + 代理配置 |
| 订阅方式 | 各种支付方式及风险说明 |
| 各服务教程 | Claude/Codex/Gemini/Cursor/国内模型 |
| 套餐选择 | $20 vs $100 组合建议 |
| 订阅规则 | 每月分享机制 + 国外模型优先原则 |

### 编写流程

每篇教程采用 **「研究-编写-验证」** 三步流程：

```text
Step A: 研究准备（WebSearch）
        → 搜索最新订阅流程、支付方式、常见问题
        → 验证价格和套餐信息
        ↓
Step B: 编写初稿
        → 按文档规范编写
        → 标注风险等级和信息来源
        ↓
Step C: 验证审查
        → 检查信息时效性
        → 用户 Review
```

## Impact

- **Affected specs**: 新增 `subscription-guide` capability
- **Affected files**: `repos/ai-programming-share/docs/` 新增 10+ markdown 文件
- **Dependencies**: 需要联网研究各服务最新订阅流程
