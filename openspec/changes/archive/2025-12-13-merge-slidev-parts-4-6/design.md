# Design: 合并 Slidev Part 4-6

## Context

AI 编程分享演讲稿（Slidev）当前有 6 个 Part，其中 Part 4-6 存在内容重叠。本次合并旨在精简结构、突出重点。

### 约束

- 保持演讲总时长 ~2 小时不变
- Part 1-3 已基本完成，不做大改动
- 重点内容必须保留：Context Engineering、OCaml 案例、现场演示

## Goals / Non-Goals

**Goals:**
- 将 Part 4-6 合并为单一 Part 4（~35-40min）
- 去除重复内容（Oxide 原则、模拟器心智等）
- 删除不合适的比喻（葵花宝典）
- 降级不熟悉的案例（Lovable）

**Non-Goals:**
- 不改动 Part 1-3 的内容
- 不添加新的主题或案例

## Decisions

### 1. 新 Part 4 的七节结构

```
4.1 心智模型（5min）
    └─ 合并「模拟器 vs 实体」心智模型，删除葵花宝典
    └─ 核心信息：问「专家会怎么说」而非「你怎么看」

4.2 Context Engineering 实操（8min）
    └─ 保留：好/坏 Context 对比
    └─ 保留：三条核心原则（有限资源、工具精简、具体胜过模糊）
    └─ 保留：Course Correct 早期纠偏

4.3 CLAUDE.md 与工具链（5min）
    └─ WHAT/WHY/HOW 框架
    └─ OpenSpec 简介（spec-driven 开发）
    └─ Skills/MCP/Slash Commands 概览

4.4 现场演示（12min）
    └─ 完整保留时钟 demo
    └─ 展示 Git Workflow Skill

4.5 使用边界（5min）
    └─ 合并「什么时候用/不用」表格
    └─ 精简「七条实践建议」
    └─ 技能退化警示（简化为 2-3 点）
    └─ 成本意识（一页 slide）

4.6 团队采用（5min）
    └─ OCaml 13K PR 案例（重点）
    └─ 分阶段采用策略
    └─ Oxide 五原则（只出现一次）
    └─ 代码审查底线

4.7 Q&A 与资源（5min）
    └─ 常见问题表格
    └─ 推荐阅读分类
```

### 2. 内容取舍原则

| 原则 | 说明 |
|------|------|
| 去重 | 同一概念只在最相关的位置出现一次 |
| 降级 | 不熟悉的内容降为脚注 |
| 删除 | 与 Part 3 重复或用词不当的内容直接删除 |
| 合并 | 相似主题的内容合并到一处 |

### 3. 脚注处理

- 保留所有 L1（原始论文/官方文档）和 L2（权威媒体）引用
- L3/L4 引用按需保留
- 统一编号，避免重复

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 合并后内容过于紧凑 | 现场演示保留 12min，给听众喘息空间 |
| 删除内容可能有价值 | 原文件 git 历史保留，需要时可恢复 |
| 脚注编号混乱 | 重新统一编号 |
